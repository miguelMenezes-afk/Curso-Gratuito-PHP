Bancos de dados e SQL básico

Entenda o que é um banco de dados relacional, como o MySQL se encaixa no ecossistema PHP e aprenda os 4 comandos SQL fundamentais que formam o CRUD.

O que é um banco de dados relacional?

Um banco de dados relacional organiza informações em tabelas — semelhante a planilhas — com linhas (registros) e colunas (campos). O MySQL é o banco relacional open-source mais popular e combina perfeitamente com PHP.

Cada tabela representa uma entidade do sistema: usuários, produtos, pedidos. As tabelas se relacionam através de chaves estrangeiras.

💡
MySQL no XAMPP

Se você instalou o XAMPP (Módulo 1), o MySQL já está incluído! Inicie o serviço MySQL no painel XAMPP e acesse o gerenciador em http://localhost/phpmyadmin.

Criando banco e tabela

1-- Criar banco de dados
2CREATE DATABASE phpmaster_db CHARACTER SET utf8mb4;
3USE phpmaster_db;
4
5-- Criar tabela de usuários
6CREATE TABLE usuarios (
7  id INT AUTO_INCREMENT PRIMARY KEY,
8  nome VARCHAR(100) NOT NULL,
9  email VARCHAR(150) UNIQUE NOT NULL,
10  ativo BOOLEAN DEFAULT 1,
11  criado_em TIMESTAMP DEFAULT CURRENT_TIMESTAMP

✅
AUTO_INCREMENT + PRIMARY KEY

O campo id INT AUTO_INCREMENT PRIMARY KEY é o padrão de ouro: o MySQL gera automaticamente um número único e crescente para cada registro inserido.

Os 4 comandos do CRUD

Todo trabalho com banco de dados se resume a 4 operações — o acrônimo CRUD:

  • Create → INSERT INTO — inserir novos registros
  • Read    → SELECT — consultar registros
  • Update → UPDATE — modificar registros existentes
  • Delete → DELETE FROM — remover registros

AULA 02 / 07⏱ ~25 min

Conectando PHP ao MySQL com PDO

PDO (PHP Data Objects) é a forma moderna e segura de conectar PHP a qualquer banco de dados. Aprenda a criar conexões robustas com boas práticas profissionais.

PDO vs MySQLi — qual escolher?

Existem duas extensões para conectar PHP ao MySQL: PDO e MySQLi. Este curso adota o PDO pelos seguintes motivos:

  • Multi-banco: PDO funciona com MySQL, PostgreSQL, SQLite e outros — MySQLi só com MySQL
  • API orientada a objetos mais limpa e moderna
  • Prepared Statements nativos, prevenindo SQL Injection desde o design
  • Adotado por frameworks como Laravel e Symfony

Criando a conexão PDO

PHP · conexao.php
1<?php
2$host = 'localhost';
3$banco = 'phpmaster_db';
4$usuario = 'root';
5$senha = ''; // Vazio no XAMPP local
6
7try {
8  $dsn = "mysql:host={$host};dbname={$banco};charset=utf8mb4";
9  $pdo = new PDO($dsn, $usuario, $senha, [
10    PDO::ATTR_ERRMODE => PDO::ERRMODE_EXCEPTION,
11    PDO::ATTR_DEFAULT_FETCH_MODE => PDO::FETCH_ASSOC,
12    PDO::ATTR_EMULATE_PREPARES => false,
13  ]);
14  echo "Conexão bem-sucedida!\n";
15} catch (PDOException $e) {
16  die("Erro: " . $e->getMessage());
17}
⚠️
Nunca exponha credenciais!

Em produção, armazene host, banco, usuário e senha em um arquivo .env ou config.php fora da pasta pública. Nunca versione esse arquivo no Git.

Exercícios Propostos

Fixando os conceitos de conexão PDO

EXERCÍCIO 01

Explique o que é o DSN (Data Source Name) no PDO. Quais informações ele contém? Por que é importante especificar charset=utf8mb4 e não apenas utf8?

// GABARITO

O DSN é uma string de conexão que identifica o banco de dados. No MySQL tem o formato: mysql:host=HOST;dbname=BANCO;charset=CHARSET. Ele contém o driver (mysql), o host do servidor, o nome do banco e a codificação.

utf8mb4 suporta o conjunto Unicode completo — incluindo emojis e caracteres de 4 bytes. O antigo utf8 do MySQL só aceita até 3 bytes e falha com emojis e certos caracteres asiáticos, gerando erros silenciosos de dados corrompidos.

EXERCÍCIO 02

Por que é recomendado envolver a criação do PDO dentro de um try/catch? O que acontece se a conexão falhar sem esse tratamento?

// GABARITO

Quando a conexão PDO falha, ela lança uma PDOException. Sem try/catch, essa exceção provoca um Fatal Error que exibe detalhes sensíveis — usuário, senha e nome do banco — diretamente na tela do usuário, criando uma falha grave de segurança.

Com try/catch você controla a mensagem exibida ao usuário (amigável e sem detalhes técnicos) e pode registrar o erro real em um arquivo de log seguro com error_log().

EXERCÍCIO 03 — DESAFIO

Escreva um arquivo database.php completo com uma função getConnection(): PDO que use o padrão Singleton, constantes de configuração, tratamento de erros adequado e retorne sempre a mesma instância PDO.

// GABARITO
<?php
// config/database.php
define('DB_HOST',    'localhost');
define('DB_NAME',    'phpmaster_db');
define('DB_USER',    'root');
define('DB_PASS',    '');
define('DB_CHARSET', 'utf8mb4');

function getConnection(): PDO {
    static $pdo = null;
    if ($pdo === null) {
        $dsn = "mysql:host=".DB_HOST.";dbname=".DB_NAME.";charset=".DB_CHARSET;
        try {
            $pdo = new PDO($dsn, DB_USER, DB_PASS, [
                PDO::ATTR_ERRMODE          => PDO::ERRMODE_EXCEPTION,
                PDO::ATTR_DEFAULT_FETCH_MODE => PDO::FETCH_ASSOC,
                PDO::ATTR_EMULATE_PREPARES => false,
            ]);
        } catch (PDOException $e) {
            error_log($e->getMessage());
            die("Serviço temporariamente indisponível.");
        }
    }
    return $pdo;
}
?>

INSERT — Criando dados

Aprenda a inserir registros no banco usando PDO com Prepared Statements — a única forma segura de enviar dados ao MySQL, prevenindo SQL Injection.

INSERT INTO com Prepared Statements

PHP · inserir.php
1<?php
2require_once 'config/database.php';
3$pdo = getConnection();
4
5$nome = "Ana Beatriz";
6$email = "ana@email.com";
7$senha = password_hash("minhasenha123", PASSWORD_BCRYPT);
8
9// 1. Preparar — placeholders :nome, :email, :senha
10$sql = "INSERT INTO usuarios (nome, email, senha) VALUES (:nome, :email, :senha)";
11$stmt = $pdo->prepare($sql);
12
13// 2. Executar com os valores reais
14$stmt->execute([
15  ':nome' => $nome,
16  ':email' => $email,
17  ':senha' => $senha,
18]);
19
20// 3. Obter o ID gerado automaticamente
21$id = $pdo->lastInsertId();
22echo "Usuário criado com ID: " . $id;
🔐
Nunca armazene senhas em texto puro!

Use sempre password_hash($senha, PASSWORD_BCRYPT) para guardar e password_verify($input, $hash) para verificar. Assim, mesmo que o banco vaze, as senhas ficam protegidas.

Por que Prepared Statements previnem SQL Injection?

SELECT — Lendo dados

Aprenda a buscar registros com SELECT, filtrar com WHERE, ordenar com ORDER BY e paginar resultados com LIMIT — tudo de forma segura com PDO.

fetch() e fetchAll() — buscando registros

PHP · listar.php
1<?php
2$pdo = getConnection();
3
4// Buscar TODOS — sem parâmetros, use query()
5$stmt = $pdo->query("SELECT * FROM produtos ORDER BY nome ASC");
6$produtos = $stmt->fetchAll(); // Array de arrays associativos
7
8foreach ($produtos as $p) {
9  echo $p['nome'] . " — R$ " . $p['preco'] . "\n";
10}
11
12// Buscar UM por ID — use prepare()
13$stmt = $pdo->prepare("SELECT * FROM produtos WHERE id = :id");
14$stmt->execute([':id' => 3]);
15$produto = $stmt->fetch(); // Retorna uma linha ou false
16
17if ($produto) {
18  echo "Encontrado: " . $produto['nome'];
19} else {
20  echo "Produto não encontrado.";
21}
💡
fetch() vs fetchAll()

fetch() retorna uma linha (ideal para busca por ID). fetchAll() retorna todas as linhas de uma vez (ideal para listagens). Para tabelas muito grandes, prefira fetch() em laço para economizar memória.

Filtros, paginação e busca por texto (LIKE)

PHP · filtros.php
1<?php
2// Busca por texto com LIKE (% é adicionado em PHP, não no SQL)
3$busca = "mouse";
4$termo = "%" . $busca . "%";
5$stmt = $pdo->prepare("SELECT * FROM produtos WHERE nome LIKE :t ORDER BY nome");
6$stmt->execute([':t' => $termo]);
7
8// Paginação com LIMIT e OFFSET
9$pagina = 1;
10$por_pagina = 10;
11$offset = ($pagina - 1) * $por_pagina;
12$sql = "SELECT * FROM produtos ORDER BY preco ASC LIMIT :lim OFFSET :off";
13$stmt = $pdo->prepare($sql);
14$stmt->bindValue(':lim', $por_pagina, PDO::PARAM_INT);
15$stmt->bindValue(':off', $offset, PDO::PARAM_INT);
16$stmt->execute();
✅
bindValue() para LIMIT e OFFSET

LIMIT e OFFSET precisam ser inteiros. Use bindValue() com PDO::PARAM_INT para garantir o tipo correto e evitar erros de sintaxe SQL.

UPDATE — Atualizando dados

Aprenda a modificar registros existentes com segurança. O UPDATE é poderoso — e perigoso sem WHERE. Veja como usá-lo corretamente e verificar quantas linhas foram afetadas.

UPDATE com PDO e rowCount()

PHP · atualizar.php
1<?php
2$pdo = getConnection();
3
4$id = 3;
5$preco = 89.90;
6$estoque = 50;
7
8$sql = "UPDATE produtos SET preco = :preco, estoque = :estoque WHERE id = :id";
9$stmt = $pdo->prepare($sql);
10$stmt->execute([':preco'=>$preco, ':estoque'=>$estoque, ':id'=>$id]);
11
12// rowCount() — quantas linhas foram realmente afetadas
13if ($stmt->rowCount() > 0) {
14  echo "Produto atualizado com sucesso!\n";
15} else {
16  echo "Nenhum produto com ID $id encontrado.\n";
17}
⚠️
SEMPRE use WHERE no UPDATE!

Um UPDATE produtos SET preco = 0 sem WHERE zera o preço de TODOS os produtos. Sempre filtre pelo ID ou outro campo único.

Soft Delete: desativar em vez de remover

Em vez de deletar registros permanentemente, o padrão profissional é usar um campo ativo ou deletado_em para "esconder" o registro sem apagá-lo:

PHP · soft-delete.php
1<?php
2// Desativar usuário (soft delete)
3$stmt = $pdo->prepare(
4  "UPDATE usuarios SET ativo = 0 WHERE id = :id"
5);
6$stmt->execute([':id' => 4]);
7echo "Usuário desativado.\n";
8
9// Na listagem, filtrar apenas os ativos:
10$stmt = $pdo->query("SELECT * FROM usuarios WHERE ativo = 1");
// ATIVIDADE INTERATIVA

Implemente uma função atualizarCampos(PDO $pdo, string $tabela, int $id, array $campos): bool que construa dinamicamente o SQL do UPDATE a partir de um array de campos. Exemplo: atualizarCampos($pdo, 'produtos', 5, ['nome'=>'Novo', 'preco'=>99.9])

// GABARITO
<?php
function atualizarCampos(PDO $pdo, string $tabela, int $id, array $campos): bool {
    if (empty($campos)) return false;

    $sets = [];
    $params = [':__id' => $id];
    foreach ($campos as $campo => $valor) {
        $sets[] = "$campo = :$campo";
        $params[":$campo"] = $valor;
    }

    $sql  = "UPDATE $tabela SET " . implode(', ', $sets) . " WHERE id = :__id";
    $stmt = $pdo->prepare($sql);
    $stmt->execute($params);
    return $stmt->rowCount() > 0;
}

Uma transação agrupa múltiplas operações em uma unidade atômica: ou tudo executa com sucesso, ou nada é aplicado. Essencial para operações críticas como cancelamento de pedidos:

17} catch (Exception $e) {
18  $pdo->rollBack(); // Desfaz TUDO se algo falhar

20}

EXERCÍCIO 03 — DESAFIO

Explique com suas palavras a diferença entre hard delete e soft delete. Em quais cenários cada abordagem é mais adequada? Cite dois sistemas reais onde o soft delete é essencial e justifique.

// GABARITO

Hard delete: Remove o registro fisicamente com DELETE FROM. O dado some para sempre. Adequado para dados temporários sem valor histórico: sessões expiradas, logs de debug, dados de teste.

Soft delete: Mantém o registro, mas o marca como "excluído" via flag (ativo=0) ou timestamp (deletado_em). O dado continua no banco, mas é filtrado nas queries normais.

Exemplos onde é essencial:

1. E-commerce: Um produto deletado que faz parte de pedidos históricos não pode ser removido — o histórico ficaria inconsistente sem a referência ao produto e seu preço na época da compra.

2. Sistemas bancários/financeiros: Regulamentações (como a LGPD e normas do BACEN) exigem que transações e registros de clientes sejam mantidos por anos. O soft delete permite cumprir essa obrigação legal sem apagar os dados.

JOIN: relacionando tabelas

O JOIN combina linhas de duas ou mais tabelas com base em uma coluna relacionada. O mais comum é o INNER JOIN, que retorna apenas registros com correspondência em ambas as tabelas:

Tipos de JOIN

INNER JOIN: só registros com correspondência em ambas · LEFT JOIN: todos da tabela esquerda + correspondências da direita (NULL quando não existe) · RIGHT JOIN: o inverso do LEFT JOIN.

Crie um CRUD completo em PHP para a tabela produtos. Implemente 4 funções: criarProduto(), listarProdutos(), atualizarProduto() e deletarProduto(). Use Prepared Statements em todas. Demonstre o uso de cada função em sequência.

// GABARITO
<?php
require_once 'config/database.php';

function criarProduto(PDO $pdo, string $nome, float $preco, int $estoque): int {
    $stmt = $pdo->prepare(
        "INSERT INTO produtos (nome, preco, estoque) VALUES (:n,:p,:e)"
    );
    $stmt->execute([':n'=>$nome,':p'=>$preco,':e'=>$estoque]);
    return (int)$pdo->lastInsertId();
}

function listarProdutos(PDO $pdo): array {
    return $pdo->query("SELECT * FROM produtos ORDER BY nome")->fetchAll();
}

function atualizarProduto(PDO $pdo, int $id, string $nome, float $preco): bool {
    $stmt = $pdo->prepare("UPDATE produtos SET nome=:n, preco=:p WHERE id=:id");
    $stmt->execute([':n'=>$nome,':p'=>$preco,':id'=>$id]);
    return $stmt->rowCount() > 0;
}

function deletarProduto(PDO $pdo, int $id): bool {
    $stmt = $pdo->prepare("DELETE FROM produtos WHERE id = :id");
    $stmt->execute([':id' => $id]);
    return $stmt->rowCount() > 0;
}
