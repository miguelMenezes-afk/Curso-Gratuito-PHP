Introdução à POO

Entenda os pilares da Programação Orientada a Objetos e por que ela revolucionou o desenvolvimento de software.

O que é POO?

A Programação Orientada a Objetos (POO) é um paradigma que organiza o código ao redor de "objetos" — entidades que combinam dados (propriedades) e comportamentos (métodos). Em vez de pensar em procedimentos e funções soltas, você modela o mundo real em código.

PHP moderno (>=5.0) suporta POO completa e é fundamental para frameworks como Laravel, Symfony e WordPress moderno.

💡
Diferença entre procedural e POO

Procedural: código organizado em funções que manipulam dados. POO: código organizado em objetos que contêm dados e comportamentos juntos.

Os 4 pilares da POO

A POO se sustenta em 4 conceitos fundamentais:

  1. Encapsulamento: ocultar detalhes internos e expor apenas o necessário
  2. Abstração: focar no essencial, ignorando detalhes irrelevantes
  3. Herança: reaproveitar código criando classes derivadas
  4. Polimorfismo: mesma interface, comportamentos diferentes

Vamos estudar cada um em detalhes nas próximas aulas.

Por que usar POO?

Vantagens da POO em projetos reais:

  • Organização: código mais limpo e estruturado
  • Reutilização: menos código duplicado
  • Manutenibilidade: mudanças isoladas, menos bugs
  • Escalabilidade: mais fácil crescer o projeto
  • Colaboração: equipes trabalham melhor em código POO
✅
Quando usar POO?

Use POO para projetos com múltiplas entidades relacionadas (usuários, produtos, pedidos...). Para scripts simples de 50 linhas, procedural pode ser suficiente.

Classes e Objetos

Aprenda a criar suas primeiras classes em PHP e instanciar objetos a partir delas.

O que é uma Classe?

Uma classe é uma "planta" ou "molde" que define a estrutura e comportamento de objetos. É como um blueprint arquitetônico: define o que será construído, mas não é a construção em si.

PHP · primeira_classe.php
1<?php
2
3class Carro {
4 // Aqui vão propriedades e métodos
5 public $marca;
6 public $modelo;
7}
8
9// Convenção: nome de classe em PascalCase
10?>

O que é um Objeto?

Um objeto é uma instância de uma classe — a "construção" feita a partir do blueprint. Você pode criar múltiplos objetos da mesma classe.

PHP · objetos.php
1<?php
2
3class Carro {
4 public $marca;
5 public $modelo;
6}
7
8// Criando objetos (instâncias)
9$carro1 = new Carro();
10$carro1->marca = "Toyota";
11$carro1->modelo = "Corolla";
12
13$carro2 = new Carro();
14$carro2->marca = "Honda";
15$carro2->modelo = "Civic";
16
17echo $carro1->marca; // Toyota
18?>

Propriedades e Métodos

Domine a estrutura interna de classes: propriedades armazenam dados, métodos definem comportamentos.

Propriedades da Classe

Propriedades (ou atributos) são variáveis que pertencem à classe. Representam as características do objeto.

Propriedades podem ter diferentes níveis de visibilidade:

  • public: acessível de qualquer lugar
  • protected: acessível apenas dentro da classe e subclasses
  • private: acessível apenas dentro da própria classe
PHP · propriedades.php
1<?php
2
3class Usuario {
4 public $nome; // Pública: acessível em todo lugar
5 protected $email; // Protegida: só classe + filhas
6 private $senha; // Privada: só nesta classe
7}
8
9$user = new Usuario();
10$user->nome = "João"; // ✓ OK (public)
11// $user->senha = "123"; // ✗ ERRO (private)
12?>

Métodos da Classe

Métodos são funções que pertencem à classe. Definem o comportamento do objeto — o que ele pode fazer.

Dentro de um método, usamos $this para acessar propriedades e outros métodos do próprio objeto.

PHP · metodos.php
1<?php
2
3class Calculadora {
4 public $resultado = 0;
5
6 public function somar($a, $b) {
7 $this->resultado = $a + $b;
8 return $this->resultado;
9 }
10
11 public function exibirResultado() {
12 echo "Resultado: " . $this->resultado;
13 }
14}
15
16$calc = new Calculadora();
17$calc->somar(10, 5);
18$calc->exibirResultado(); // Resultado: 15
19?>
🔑
A palavra-chave $this

$this é uma variável especial que referencia o próprio objeto dentro de um método. Use-a para acessar propriedades e chamar outros métodos.

Pratique propriedades, métodos e $this

EXERCÍCIO 01

Qual a diferença entre uma propriedade e um método em uma classe?

// GABARITO

Propriedade: armazena dados/características do objeto (ex: $nome, $idade, $preco).

Método: define comportamentos/ações que o objeto pode executar (ex: calcular(), salvar(), enviar()).

Resumo: propriedades = dados, métodos = funções.

EXERCÍCIO 02

Crie uma classe ContaBancaria com propriedade $saldo e métodos depositar($valor) e sacar($valor). Teste depositando R$100 e sacando R$30.

// GABARITO
<?php
class ContaBancaria {
  public $saldo = 0;

  public function depositar($valor) {
    $this->saldo += $valor;
    echo "Depósito: R$ $valor\n";
  }

  public function sacar($valor) {
    if ($this->saldo >= $valor) {
      $this->saldo -= $valor;
      echo "Saque: R$ $valor\n";
    } else {
      echo "Saldo insuficiente!\n";
    }
  }
}

$conta = new ContaBancaria();
$conta->depositar(100);  // Depósito: R$ 100
$conta->sacar(30);       // Saque: R$ 30
echo "Saldo final: R$ " . $conta->saldo; // R$ 70
?>
EXERCÍCIO 03

Por que usamos $this dentro de métodos? O que acontece se não usarmos?

// GABARITO

Usamos $this para referenciar propriedades e métodos do próprio objeto dentro da classe.

Sem $this: PHP trata como variável local do método, não como propriedade da classe. Ex: $nome (variável local) vs $this->nome (propriedade da classe).

AULA 04 / 08 ⏱ ~20 min

Construtores

Aprenda a inicializar objetos automaticamente com o método mágico __construct().

O que é um Construtor?

O construtor é um método especial que é executado automaticamente sempre que um novo objeto é criado. Serve para inicializar propriedades.

Em PHP, o construtor é definido com o nome __construct() (dois underscores).

PHP · construtor.php
1<?php
2
3class Pessoa {
4 public $nome;
5 public $idade;
6
7 // Construtor
8 public function __construct($nome, $idade) {
9 $this->nome = $nome;
10 $this->idade = $idade;
11 }
12}
13
14// Ao criar o objeto, o construtor é chamado
15$p1 = new Pessoa("Ana", 25);
16echo $p1->nome; // Ana
17?>
✨
Vantagem do construtor

Sem construtor, você precisa definir cada propriedade manualmente após criar o objeto. Com construtor, tudo é feito em uma linha!

Construtor com valores padrão

Você pode definir valores padrão para parâmetros do construtor, tornando-os opcionais.

PHP · construtor_padrao.php
1<?php
2
3class Config {
4 public $tema;
5 public $idioma;
6
7 public function __construct($tema = "claro", $idioma = "pt-BR") {
8 $this->tema = $tema;
9 $this->idioma = $idioma;
10 }
11}
12
13$cfg1 = new Config(); // tema=claro, idioma=pt-BR
14$cfg2 = new Config("escuro"); // tema=escuro, idioma=pt-BR
15$cfg3 = new Config("escuro", "en-US"); // tema=escuro, idioma=en-US
16?>

Herança

Reaproveite código criando classes filhas que herdam características de classes pai.

O que é Herança?

Herança permite criar uma nova classe (filha) baseada em uma classe existente (pai). A classe filha herda todas as propriedades e métodos públicos/protegidos da classe pai.

Usa-se a palavra-chave extends para estabelecer herança.

PHP · heranca.php
1<?php
2
3// Classe PAI
4class Animal {
5 public $nome;
6
7 public function comer() {
8 echo $this->nome . " está comendo\n";
9 }
10}
11
12// Classe FILHA
13class Cachorro extends Animal {
14 public function latir() {
15 echo "Au au!\n";
16 }
17}
18
19$dog = new Cachorro();
20$dog->nome = "Rex";
21$dog->comer(); // Herdado de Animal
22$dog->latir(); // Próprio de Cachorro
23?>

Sobrescrita de métodos

A classe filha pode sobrescrever (override) métodos da classe pai, fornecendo sua própria implementação.

PHP · override.php
1<?php
2
3class Forma {
4 public function desenhar() {
5 echo "Desenhando uma forma...\n";
6 }
7}
8
9class Circulo extends Forma {
10 // Sobrescreve o método da classe pai
11 public function desenhar() {
12 echo "Desenhando um círculo ⭕\n";
13 }
14}
15
16$forma = new Forma();
17$forma->desenhar(); // Desenhando uma forma...
18
19$circulo = new Circulo();
20$circulo->desenhar(); // Desenhando um círculo ⭕
21?>

A palavra-chave parent::

Use parent::metodo() para chamar a versão original do método da classe pai dentro de um método sobrescrito.

Encapsulamento

Proteja seus dados usando visibilidade de propriedades e métodos getters/setters.

O que é Encapsulamento?

Encapsulamento é o princípio de ocultar detalhes internos de uma classe e expor apenas o necessário. Isso protege dados contra modificações indevidas.

Usamos modificadores de acesso para controlar visibilidade:

  • public: acessível de qualquer lugar
  • protected: acessível na classe e subclasses
  • private: acessível apenas dentro da própria classe
PHP · encapsulamento.php
1<?php
2
3class ContaBancaria {
4 private $saldo; // Oculto do exterior
5
6 public function __construct($saldoInicial) {
7 $this->saldo = $saldoInicial;
8 }
9
10 public function depositar($valor) {
11 if ($valor > 0) {
12 $this->saldo += $valor;
13 }
14 }
15}
16
17$conta = new ContaBancaria(1000);
18// $conta->saldo = -500; // ✗ ERRO: saldo é private
19$conta->depositar(200); // ✓ OK: usa método público
20?>
🛡️
Por que encapsular?

Se $saldo fosse público, alguém poderia fazer $conta->saldo = -999999! Com encapsulamento, controlamos como os dados são modificados.

Getters e Setters

Getters (métodos get) permitem ler propriedades privadas. Setters (métodos set) permitem modificar propriedades privadas com validação.

PHP · getters_setters.php
1<?php
2
3class Pessoa {
4 private $idade;
5
6 // SETTER: define valor com validação
7 public function setIdade($idade) {
8 if ($idade >= 0 && $idade <= 150) {
9 $this->idade = $idade;
10 } else {
11 echo "Idade inválida!\n";
12 }
13 }
14
15 // GETTER: retorna o valor
16 public function getIdade() {
17 return $this->idade;
18 }
19}
20
21$p = new Pessoa();
22$p->setIdade(25); // ✓ Válido
23$p->setIdade(999); // ✗ Idade inválida!
24echo $p->getIdade(); // 25
25?>

Interfaces

Defina contratos que classes devem cumprir usando interfaces.

O que é uma Interface?

Uma interface é um contrato que define quais métodos uma classe deve implementar, mas não como. É como um esqueleto de métodos sem corpo.

Interfaces garantem que diferentes classes sigam o mesmo padrão, facilitando substituição e polimorfismo.

PHP · interface.php
1<?php
2
3// Define o contrato
4interface Pagavel {
5 public function calcularPagamento();
6}
7
8// Implementa a interface
9class Funcionario implements Pagavel {
10 public $salario = 3000;
11
12 public function calcularPagamento() {
13 return $this->salario;
14 }
15}
16
17class Freelancer implements Pagavel {
18 public $horasTrabalhadas = 40;
19 public $valorHora = 100;
20
21 public function calcularPagamento() {
22 return $this->horasTrabalhadas * $this->valorHora;
23 }
24}
25
26$f = new Funcionario();
27echo $f->calcularPagamento(); // 3000
28?>
📋
Interface vs Classe Abstrata

Interface: apenas métodos sem implementação. Uma classe pode implementar múltiplas interfaces.
Classe abstrata: pode ter métodos implementados. Uma classe só pode herdar de uma classe abstrata.

Múltiplas Interfaces

Uma classe pode implementar várias interfaces ao mesmo tempo, separando-as por vírgula.

PHP · multiplas_interfaces.php
1<?php
2
3interface Voador {
4 public function voar();
5}
6
7interface Nadador {
8 public function nadar();
9}
10
11class Pato implements Voador, Nadador {
12 public function voar() {
13 echo "Pato voando! 🦆\n";
14 }
15
16 public function nadar() {
17 echo "Pato nadando! 🦆\n";
18 }
19}
20
21$donald = new Pato();
22$donald->voar();
23$donald->nadar();
24?>

Pratique criação e uso de interfaces

EXERCÍCIO 01

Qual a principal diferença entre uma interface e uma classe abstrata?

// GABARITO

Interface:

  • Apenas métodos sem implementação (apenas assinaturas)
  • Uma classe pode implementar múltiplas interfaces
  • Não pode ter propriedades

Classe Abstrata:

  • Pode ter métodos implementados e não implementados
  • Uma classe só pode herdar de uma classe abstrata (herança única)
  • Pode ter propriedades

EXERCÍCIO 03

Por que interfaces são úteis para polimorfismo? Dê um exemplo prático.

// GABARITO

Interfaces permitem tratar objetos diferentes de forma uniforme se implementam a mesma interface.

Exemplo: Você tem um array de objetos Pagavel (interface). Esse array pode conter Funcionario, Freelancer, Fornecedor... Todos diferentes, mas você pode fazer um loop e chamar calcularPagamento() em todos — polimorfismo em ação!

Namespaces

Organize código em projetos grandes evitando conflitos de nomes com namespaces.

O que são Namespaces?

Namespaces são como "pastas" para organizar classes e funções. Resolvem o problema de duas classes terem o mesmo nome em partes diferentes do código.

Imagine que você tem uma classe Database no seu código e outra biblioteca também tem uma classe Database. Namespaces evitam esse conflito.

PHP · namespace.php
1<?php
2
3// Definindo um namespace
4namespace App\Models;
5
6class Usuario {
7 public $nome = "João";
8}
9
10// Usando a classe
11$user = new Usuario();
12echo $user->nome;
13?>
📁
Convenção de namespace

Por convenção, namespaces seguem a estrutura de pastas. Ex: arquivo em App/Models/Usuario.php teria namespace App\Models.

Usando classes de outros namespaces

Para usar uma classe de outro namespace, você pode:

  • Usar o caminho completo (fully qualified name)
  • Importar com use
PHP · uso_namespace.php
1<?php
2
3// Método 1: Caminho completo
4$user1 = new \App\Models\Usuario();
5
6// Método 2: Importar com use (recomendado)
7use App\Models\Usuario;
8
9$user2 = new Usuario(); // Mais limpo!
10
11// Método 3: Alias (apelido)
12use App\Models\Usuario as User;
13$user3 = new User();
14?>

Em frameworks modernos (Laravel, Symfony), namespaces são essenciais:

14 }
15}
16?>

Em projetos modernos, usa-se Composer para autoloading. O Composer carrega automaticamente classes baseado no namespace — você não precisa fazer require manual!

EXERCÍCIO 01

Por que namespaces são importantes em projetos grandes com múltiplas bibliotecas?

// GABARITO

Razões principais:

  • Evita conflitos: Duas bibliotecas podem ter classes com mesmo nome sem conflito
  • Organização: Agrupa classes relacionadas logicamente
  • Legibilidade: Fica claro de onde vem cada classe
  • Manutenção: Facilita encontrar e modificar código
EXERCÍCIO 02

Explique a diferença entre estas duas formas de usar uma classe:
new \App\Models\Usuario() vs use App\Models\Usuario; new Usuario()

// GABARITO

1ª forma (caminho completo): Usa o nome totalmente qualificado direto. Funciona, mas fica verboso se usado muitas vezes.

2ª forma (com use): Importa a classe no topo do arquivo, depois pode usar o nome curto. Recomendado! Código mais limpo e legível.

Ambas funcionam, mas use é a prática padrão em PHP moderno.

EXERCÍCIO 03

Em um projeto Laravel, você vê namespace App\Http\Controllers; no início de um arquivo. O que isso significa?

// GABARITO

Significa que todas as classes definidas nesse arquivo pertencem ao namespace App\Http\Controllers.

Isso indica:

  • O arquivo provavelmente está em app/Http/Controllers/
  • Contém controllers (controladores) da aplicação
  • Para usar essas classes em outro lugar: use App\Http\Controllers\NomeController;
